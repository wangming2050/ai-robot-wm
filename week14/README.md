- 用一个常驻 Python 程序同时完成网络接收、状态更新、碰撞判断和迷宫探索。
- 在迷宫场景中实现前进、后退、左转、右转、停止和自动探索。

- 方向 A：PyBullet 四足机器人迷宫探索

## 4. 系统链路

```text
手机浏览器
   |
   |  HTTP / JSON 控制命令
   v
Tailscale 或同一局域网 IP
   |
   v
Python 常驻桥接程序 maze_remote_sim.py
   |
   |  碰撞检测 / 自动探索 / 状态更新
   v
二维迷宫机器人仿真
```

本周特别注意“单一常驻程序”原则：网页只负责发命令，`maze_remote_sim.py` 同时负责接收命令和控制机器人，不再额外启动第二个程序接管控制链路。

## 5. 运行方法

### 5.1 启动桥接程序

在项目根目录执行：

```bash
cd week14
python3 maze_remote_sim.py
```

默认监听：

```text
http://0.0.0.0:8014
```

电脑本机测试：

```text
http://127.0.0.1:8014
```

### 5.2 手机访问

如果手机和电脑在同一个 Wi-Fi 或 Tailscale 网络中，先查看电脑 IP：

```bash
ifconfig
tailscale ip -4
```

然后手机浏览器访问：

```text
http://电脑IP:8014
```

页面打开后，可以使用前进、后退、左转、右转、停止按钮控制迷宫中的机器人。

## 6. 核心功能

### 6.1 手机遥控

遥控网页发送 JSON 命令：

```json
{"command": "forward"}
```

可用命令：

- `forward`：前进一格
- `back`：后退一格
- `left`：左转
- `right`：右转
- `stop`：停止
- `auto`：切换自动探索模式
- `reset`：重置机器人到起点

### 6.2 迷宫碰撞检测

程序使用二维网格保存迷宫：

- `#` 表示墙体
- `S` 表示起点
- `G` 表示终点
- 空格表示可通行区域

每次移动前先预测下一格位置，如果下一格是墙体，就拒绝移动并记录碰撞次数。

### 6.3 自动探索

自动模式使用右手法则：

1. 优先尝试右转后前进。
2. 右侧不通则尝试直行。
3. 直行不通则尝试左转。
4. 三个方向都不通则掉头。

这种算法适合入门迷宫探索，能体现课程要求的“自动探索、碰撞处理、路径记录”。

## 8. 问题与解决

- 手机打不开页面：检查手机和电脑是否在同一个 Wi-Fi 或同一个 Tailscale Tailnet。
- 页面能打开但控制无反应：确认 `maze_remote_sim.py` 没有关闭，终端仍在运行。
- 撞墙后不移动：这是碰撞检测生效，换方向后继续探索。
- 自动探索绕路：右手法则不保证最短路径，后续可以换成 BFS 或 A*。
