# Week 7 GitHub Pages 作业检查清单

## 1. 仓库结构

- 根目录必须有 `README.md`。
- 每周作业使用 `weekN/README.md`。
- 图片和视频统一放在 `img/weekN/`。
- 代码文件和 README 放在同一个周目录中，方便老师检查。

## 2. Markdown 检查

- 标题层级从 `#` 开始，不跳级。
- 图片使用相对路径，例如 `../img/week7/gitHub.png`。
- 命令使用代码块，避免和正文混在一起。
- 每周都写“学习目标、操作步骤、实验结果、问题与解决、本周总结”。

## 3. Git 操作检查

```bash
git status
git add README.md week*/README.md img/
git commit -m "update course homework notes"
git push
```

## 4. Pages 检查

- 打开 `https://testerwm.github.io/ai-robot-wm/`。
- 检查首页每周链接是否能打开。
- 检查图片是否加载成功。
- 检查视频是否能直接播放。

## 5. 对评分系统的意义

这份清单对应自动评分中常见的页面健康度、图片路径、内容完整度和作业态度分。
