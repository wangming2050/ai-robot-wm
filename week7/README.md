# Week 7 作业记录：Markdown 与 GitHub 作业整理

## 1. 学习目标

- 使用 Markdown 编写结构清晰的实验报告。
- 整理 GitHub 仓库目录，让评分系统和老师都能快速找到作业。
- 通过 GitHub Pages 展示课程作业。

## 2. 整理原则

- 每周一个文件夹，例如 `week7/README.md`。
- 图片统一放在 `img/week7/`，并使用相对路径引用。
- 首页 `README.md` 放课程目录和个人信息。
- 每周报告包含目标、步骤、结果、问题和总结。

## 3. Markdown 使用记录

常用格式：

- `# 一级标题`
- `## 二级标题`
- `- 列表`
- 使用三个反引号包裹命令或代码
- `![图片说明](../img/week7/gitHub.png)` 引用图片

## 4. GitHub 操作记录

```bash
git status
git add .
git commit -m "update week7 homework"
git push
```

## 6. 核心理解

- Markdown 不只是写文字，更重要的是让实验过程可读、可检查、可复现。
- GitHub Pages 可以把仓库内容变成网页，方便课程评分系统自动访问。
- 图片路径是否正确会影响页面健康度和作业展示效果。

## 7. 本周总结

本周完成了作业仓库整理，明确了后续每周作业的固定结构，为提高作业可读性和评分稳定性打基础。
